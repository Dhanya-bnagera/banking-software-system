# banking-software-system
Overview:
The Banking Software System is a mini project developed as part of the Software Project Planning and Management assignment. It demonstrates the complete Software Development Life Cycle (SDLC) using Jira, Confluence, and GitHub.
The system allows a user to perform basic banking operations such as:
Secure login
Account balance inquiry
Deposit and withdrawal
Money transfer
View transaction history

Project Objectives:
To simulate a real-world banking operations workflow.
To use Jira for task management and issue tracking.
To use Confluence for project documentation and planning.
To use GitHub for version control and collaboration.
To apply software effort estimation and costing techniques.

Project Scope:
In Scope:
Login authentication
Balance inquiry
Deposit and withdrawal
Internal bank transfer
Transaction record storage
Out of Scope:
Loan module
ATM card/UPI payments
Mobile banking app
External bank transfers

Requirements:
Functional Requirements:
Login and authentication
Restaurant search
Order checkout
Non-Functional Requirements:
Fast response time
Secure transactions
Scalable system

Summary:
This project will simulate a real-world banking application using proper SDLC practices and industry tools (Jira, GitHub, Confluence). The results will demonstrate planning, version control, team coordination, and software cost estimation.
